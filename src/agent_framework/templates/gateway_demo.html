Gateway + WebSocket 联调页

这个页面用于验证新接入的 Gateway 长连接层：连接鉴权、在线状态、心跳、chat.send、主动推送、ACK 回写、离线补发。

1. 连接 Gateway

优先填 JWT Token；开发联调阶段也可仅填 user_id（当前后端保留 fallback）。
尚未连接

2. 通过 WebSocket 发起 chat.send

行为说明
  • 页面会向 /gateway namespace 发送 chat.send
  • 服务端内部调用现有 unified orchestrator
  • 回答将通过 chat.message 回推到当前连接
  • 客户端收到后会自动回 ACK

3. 通过 HTTP 主动推送

POST /api/gateway/push { "user_id": "u1001", "event": "notify.system", "target": "ALL", "payload": { "title": "任务完成", "content": "AI 诊断已完成" } }

4. 联调提示

推荐流程
  • 先连接 Gateway，确认在线
  • 调用 /api/gateway/online-users 检查在线用户
  • 再调用主动推送接口验证单播/多端同步
  • 最后断开连接，再推送一次，验证离线补发
常用接口
  • GET /api/gateway/nodes
  • GET /api/gateway/online-users
  • GET /api/gateway/users/<user_id>/connections
  • GET /api/gateway/users/<user_id>/offline-events
  • GET /api/gateway/events/<event_id>
